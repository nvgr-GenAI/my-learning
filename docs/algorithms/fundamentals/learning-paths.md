## üìö Experience-Based Paths

=== "Beginner Path (4-8 weeks)"
    **Profile:** New to algorithms or programming concepts

    **Prerequisites:** Basic programming knowledge in any language

    **Time Commitment:** 10-15 hours/week

    | Week | Topics | Practice |
    |------|--------|----------|
    | **1-2** | Arrays, Strings, Hash Tables | 10-15 easy problems |
    | **3-4** | Linked Lists, Stacks, Queues | 10-15 easy problems |
    | **5-6** | Basic Sorting, Binary Search | 5-10 implementation problems |
    | **7-8** | Trees, Basic Recursion, Graph Intro | 10-15 easy problems |

    **Study Order:**
    1. [Arrays](../data-structures/arrays/index.md) ‚Üí [Hash Tables](../data-structures/hash-tables/index.md)
    2. [Linked Lists](../data-structures/linked-lists/index.md) ‚Üí [Stacks](../data-structures/stacks/index.md) ‚Üí [Queues](../data-structures/queues/index.md)
    3. [Sorting](../sorting/index.md) ‚Üí [Binary Search](../searching/binary-search.md)
    4. [Trees](../trees/index.md) ‚Üí [Graphs](../graphs/index.md)

    **Success Metric:** Solve easy problems confidently, understand basic complexity analysis

=== "Intermediate Path (8-12 weeks)"
    **Profile:** Familiar with basic data structures and algorithms

    **Prerequisites:** Completed beginner path or equivalent knowledge

    **Time Commitment:** 10-15 hours/week

    | Week | Topics | Practice |
    |------|--------|----------|
    | **1-3** | Heaps, BST, Advanced Trees | 15-20 medium problems |
    | **4-6** | Graph Algorithms (BFS, DFS, Dijkstra) | 15-20 medium problems |
    | **7-9** | Dynamic Programming Basics | 15-20 medium problems |
    | **10-12** | Greedy, Two Pointers, Sliding Window | 15-20 medium problems |

    **Study Order:**
    1. [Heaps](../data-structures/heaps/index.md) ‚Üí [BST Operations](../trees/index.md)
    2. [Graph Traversals](../graphs/index.md) ‚Üí [Shortest Paths](../graphs/dijkstra.md)

    **Success Metric:** Solve medium problems, optimize solutions, explain complexity

## üéØ Goal-Based Tracks

=== "Interview Prep Track"
    **Target:** FAANG, tech companies, startups

    **Timeline:** 2-3 months intensive

    **Focus:** Problem patterns, complexity analysis, communication

    **Must-Master Topics:**

    | Priority | Topic | Why Important |
    |----------|-------|---------------|
    | üî• High | Arrays, Hash Tables, Trees | 60% of interview problems |
    | üî• High | Two Pointers, Sliding Window | Common optimization patterns |
    | üî• High | BFS, DFS, Binary Search | Fundamental algorithms |
    | üü° Medium | Dynamic Programming | 20% of problems, shows problem-solving |
    | üü° Medium | Graphs (Dijkstra, Topological Sort) | Common in system design |
    | üü¢ Low | Advanced Trees (Segment Tree, Trie) | Rare but impressive |

    **Weekly Plan:**
    - **Week 1-4:** Master data structures (arrays through trees)
    - **Week 5-8:** Learn problem patterns + 40-50 medium problems
    - **Week 9-10:** Dynamic programming + 15-20 DP problems
    - **Week 11-12:** Mock interviews + system design basics

    **Practice Mix:** 60% Medium, 30% Easy, 10% Hard

    **Resources:**
    - [Interview Strategy](interview-strategy.md)
    - [Problem-Solving Patterns](problem-solving-patterns.md)
    - [LeetCode Top Interview 150](https://leetcode.com/studyplan/top-interview-150/)

=== "Competition Track"
    **Target:** CodeForces, ICPC, Google Code Jam

    **Timeline:** 4-6 months intensive

    **Focus:** Speed, mathematical foundations, advanced algorithms

    **Core Curriculum:**

    | Phase | Topics | Skill Level |
    |-------|--------|-------------|
    | **Phase 1** (1-2 months) | All fundamental DS + algorithms | Solve Div 2 A-C consistently |
    | **Phase 2** (2-3 months) | Advanced graphs, DP optimization, Number theory | Solve Div 2 D-E occasionally |
    | **Phase 3** (3-4 months) | Segment trees, FFT, Advanced strings | Solve Div 1 A-B consistently |
    | **Phase 4** (4-6 months) | Geometry, Game theory, Specialized DS | Compete at Div 1 level |

    **Key Differences from Interview Prep:**
    - Focus on speed (implement quickly)
    - Mathematical problem-solving
    - Advanced data structures (Fenwick, Segment Trees)
    - Template library development

    **Resources:**
    - [CP-Algorithms](https://cp-algorithms.com/)
    - [USACO Guide](https://usaco.guide/)
    - [CodeForces](https://codeforces.com/)

=== "Engineering Track"
    **Target:** Backend/fullstack developers, software engineers

    **Timeline:** 2-4 months balanced

    **Focus:** Practical implementations, real-world applications, system design

    **Practical Curriculum:**

    | Area | Topics | Application |
    |------|--------|-------------|
    | **Core DS** | Hash Maps, Trees, Graphs, Queues | Caching, routing, task scheduling |
    | **Algorithms** | Sorting, Search, Graph traversal | Data processing, web crawling |
    | **System Design** | Scalability, databases, caching | Building production systems |
    | **Performance** | Complexity analysis, profiling | Optimization, bottleneck identification |

    **Practical Projects:**
    - Build a cache system (LRU, TTL)
    - Implement a web crawler (BFS/DFS)
    - Design a rate limiter (sliding window)
    - Create a task scheduler (priority queue)

    **Resources:**
    - [System Design Guide](../../system-design/index.md)
    - [Time Complexity](time-complexity.md)
    - GitHub open source projects

## üìä Practice Strategy

=== "Problem Selection"
    **Progression Framework:**

    | Stage | Problem Type | Success Rate | What to Do |
    |-------|--------------|--------------|------------|
    | **Building** | Easy | < 70% | Focus on fundamentals, slow down |
    | **Practicing** | Medium | 50-70% | Perfect stage - challenging but achievable |
    | **Mastering** | Hard | 30-50% | Push boundaries, learn new techniques |
    | **Too Hard** | Any | < 30% | Step back, fill knowledge gaps |

    **Quality over Quantity:** Better to solve 100 problems deeply than 300 superficially

=== "Study Techniques"
    | Technique | How to Apply | Benefit |
    |-----------|--------------|---------|
    | **Spaced Repetition** | Revisit problems after 1 day, 3 days, 1 week, 1 month | Long-term retention |
    | **Active Recall** | Code from memory without looking | Strengthen understanding |
    | **Elaboration** | Explain solution out loud or in writing | Deepen comprehension |
    | **Interleaving** | Mix problem types instead of blocking | Pattern recognition |
    | **Mock Interviews** | Simulate real interview pressure | Communication skills |

=== "When You're Stuck"
    **Struggle Threshold:** Spend 25-30 minutes before looking at hints

    **Progressive Help:**
    1. Read problem again carefully (missed constraint?)
    2. Draw examples on paper (see pattern?)
    3. Consider brute force first (optimize later)
    4. Look at hint (not full solution)
    5. Study solution, understand why it works
    6. Implement from memory next day

## üí° Success Principles

!!! success "Keys to Mastery"
    1. **Consistency > Intensity** - 1 hour daily beats 8 hours on weekends
    2. **Understanding > Memorization** - Know why, not just how
    3. **Patterns > Problems** - Recognize categories, not individual solutions
    4. **Quality > Quantity** - 10 problems deeply > 50 problems superficially
    5. **Communication** - Practice explaining your thought process aloud

!!! warning "Common Mistakes to Avoid"
    - Jumping to hard problems too quickly
    - Looking at solutions immediately when stuck
    - Not reviewing problems after solving
    - Ignoring complexity analysis
    - Practicing only one type of problem

!!! quote "Remember"
    > "The only way to learn algorithms is by solving problems." ‚Äî Donald Knuth (paraphrased)

    Progress isn't linear. Plateaus are normal. Persistence and consistent practice will get you there.
