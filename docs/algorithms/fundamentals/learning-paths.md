    ## Beginner Path

    **Profile:**
    - New to algorithms or programming
    - Completed basic programming course (variables, loops, functions)
    - Want to build strong foundations

    **Timeline:** 4-8 weeks
    **Time commitment:** 10-15 hours/week
    **Total problems:** 60-80 (mostly easy)

    | Week | Topics | Learn | Practice | Milestone |
    |------|--------|-------|----------|-----------|
    | **1-2** | Arrays, Strings, Hash Tables | Two pointers, frequency counting | 15 easy | Solve Two Sum variants |
    | **3-4** | Linked Lists, Stacks, Queues | Pointer manipulation, LIFO/FIFO | 15 easy | Reverse linked list from memory |
    | **5-6** | Sorting, Binary Search | Comparison sorts, divide & conquer | 10 easy | Implement merge sort |
    | **7-8** | Trees, Recursion, Graphs | Traversals, base cases, BFS/DFS | 15 easy | Tree traversals without looking |

    **Daily Routine:**
    ```
    Day 1-3: Theory (1h) + 1 easy problem (1h) = 2h
    Day 4-6: Review (0.5h) + 2 easy problems (2h) = 2.5h
    Day 7: Weekly review (1h) + Mock practice (1h) = 2h
    ```

    **Success Criteria:**
    - [ ] Solve 80% of easy problems without hints
    - [ ] Explain time/space complexity of your solutions
    - [ ] Implement common data structures from scratch
    - [ ] Debug code independently

    **Next Steps:** ‚Üí [Intermediate Path](#intermediate-path) or [Interview Track](#interview-prep-track)

    ## Competition Track

    **Target:** Codeforces, ICPC, Google Code Jam, TopCoder
    **Timeline:** 4-6 months
    **Focus:** Speed, mathematics, advanced algorithms

    ### Skill Progression

    | Phase | Duration | Rating Target | Focus Areas |
    |-------|----------|---------------|-------------|
    | **Beginner** | Month 1-2 | CF 1200-1400 | All fundamentals, Div 2 A-B |
    | **Intermediate** | Month 2-4 | CF 1400-1600 | Advanced DS, Div 2 C-D |
    | **Advanced** | Month 4-6 | CF 1600-1800 | Math, DP opt, Div 2 E / Div 1 A-B |
    | **Expert** | Month 6+ | CF 1800+ | Specialized topics, Div 1 C-D |

    ### Core Curriculum

    === "Phase 1: Foundations (Month 1-2)"

        **Data Structures:**
        - Arrays, hash tables, sets
        - Stacks, queues, deques
        - Binary trees, BST, heaps
        - Disjoint sets (Union-Find)

        **Algorithms:**
        - Binary search and variants
        - Two pointers, sliding window
        - BFS, DFS, shortest paths
        - Basic DP (1D, 2D)

        **Practice:** 50-60 problems (Div 2 A-B difficulty)

    === "Phase 2: Advanced Structures (Month 2-4)"

        **Data Structures:**
        - Segment trees, Fenwick trees
        - Trie, suffix arrays
        - Sparse tables
        - Policy-based DS (ordered set)

        **Algorithms:**
        - Advanced DP (bitmask, digit, SOS)
        - Graph algorithms (SCC, bridges, articulation points)
        - Number theory (GCD, modular arithmetic, prime sieves)
        - Greedy with proof

        **Practice:** 60-70 problems (Div 2 C-D difficulty)

    === "Phase 3: Specialization (Month 4-6)"

        **Topics:**
        - String algorithms (KMP, Z, hashing, suffix automation)
        - Math (FFT, NTT, combinatorics, probability)
        - Geometry (convex hull, line sweep)
        - Game theory (Nim, Grundy numbers)
        - Flow algorithms (max flow, min cost flow)

        **Practice:** 50-60 problems (Div 2 E / Div 1 A-B difficulty)

    ### Training Routine

    **Daily (2-3 hours):**
    - Solve 1-2 problems just above your level
    - Implement one algorithm from template
    - Read editorial of unsolved problems

    **Weekly:**
    - Participate in 2 contests (Codeforces, AtCoder, etc.)
    - Upsolve 3-5 problems you couldn't solve in contest
    - Review and update algorithm template library

    **Monthly:**
    - Analyze rating progression
    - Identify weak topics
    - Deep dive into 1-2 new advanced topics

=== "üìä Practice Strategies"

    ## Problem Selection Framework

    ### Difficulty Progression

    **Follow the 70% rule:** You should be solving ~70% of problems at your current level.

    | Success Rate | Action | What It Means |
    |--------------|--------|---------------|
    | **< 50%** | Step back to easier problems | Too difficult, hurting confidence |
    | **50-70%** | Perfect! Stay at this level | Optimal learning zone |
    | **70-85%** | Mix in harder problems | Ready to level up |
    | **> 85%** | Move to next difficulty | Too easy, wasting time |

    ### Study Techniques

    === "Spaced Repetition"

        **Proven method for long-term retention**

        | Review | After Initial Solve | What to Do |
        |--------|---------------------|------------|
        | **1st** | +1 day | Solve again from memory |
        | **2nd** | +3 days | Solve similar problem |
        | **3rd** | +1 week | Explain solution aloud |
        | **4th** | +2 weeks | Solve harder variant |
        | **5th** | +1 month | Teach someone else |

        **Tool:** Anki flashcards, spreadsheet tracker

    === "Active Recall"

        **Force your brain to retrieve information**

        **Instead of:**
        - Re-reading code
        - Looking at solution again
        - Copying and pasting

        **Do:**
        - Code from memory (blank screen)
        - Explain to rubber duck
        - Draw solution on paper first

    === "Deliberate Practice"

        **Focus on weaknesses, not strengths**

        ```
        1. Identify weak area (e.g., DP, graph algorithms)
        2. Study theory (1-2 hours)
        3. Solve 5-10 problems in that area
        4. Review mistakes deeply
        5. Re-test in 1 week
        ```

        **Example:** Struggled with DP?
        - Day 1: Study DP patterns (2h)
        - Day 2-6: Solve 10 DP problems (1-2h daily)
        - Day 7: Mock interview with DP problems
        - Day 14: Revisit same 10 problems

    === "Interleaving"

        **Mix problem types instead of blocking**

        **Blocked Practice** (less effective):
        ```
        Day 1-3: Only array problems (10 problems)
        Day 4-6: Only tree problems (10 problems)
        Day 7-9: Only DP problems (10 problems)
        ```

        **Interleaved Practice** (more effective):
        ```
        Day 1: 1 array + 1 tree + 1 DP
        Day 2: 1 tree + 1 DP + 1 graph
        Day 3: 1 DP + 1 array + 1 tree
        ...
        ```

        **Why:** Forces pattern recognition, mimics real interviews

    ## When You're Stuck

    ### The 30-Minute Rule

    ```
    0-10 min: Understand problem, try brute force
    10-20 min: Consider patterns, optimize approach
    20-30 min: Start implementation or seek hint
    30+ min: Look at hint (not full solution)
    ```

    ### Progressive Help System

    | Level | What to Use | When |
    |-------|-------------|------|
    | **Hint 1** | Re-read constraints | After 5 min |
    | **Hint 2** | Draw examples, look for pattern | After 15 min |
    | **Hint 3** | Google "[pattern] + problem type" | After 25 min |
    | **Hint 4** | Read approach (not code) | After 30 min |
    | **Solution** | Study editorial with code | After honest attempt |

    ### After Viewing Solution

    **Don't just move on! Follow up:**

    1. Close the solution
    2. Implement from memory (same day)
    3. Solve again next day (no peeking)
    4. Find similar problem, solve independently
    5. Add to spaced repetition schedule
